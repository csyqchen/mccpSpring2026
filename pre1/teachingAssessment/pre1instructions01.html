Presentation 1: Student Instructions

MCCP6020 — Oral Presentation Assessment 1: Research Story-telling by Experienced Writers (20%)

What You Need to Do — Overview

You will work with an AI agent (e.g., Cursor) to prepare an 8-minute presentation on an empirical journal article in your field. The AI generates your presentation outline and HTML slides — but you review, give feedback, and practice the delivery yourself.

Important: Reading from a script is not permitted and will result in a failing grade. Note cards with key points are allowed.

Step 0: Pull the Latest Repository Changes

You have already forked https://github.com/tesolchina/mccpSpring2026. Now you need to sync your fork to get the latest files.

Open your AI agent (Cursor) and give it this instruction:

Pull the latest changes from the upstream repository https://github.com/tesolchina/mccpSpring2026 into my local clone and push to my fork.

After syncing, navigate to your working folder: pre1/teachingAssessment/pre1work/

Step 1: Choose and Place Your Paper

Select an empirical journal article in your field and place it in pre1work/papers/.

File format requirement: Your paper must be in one of these formats:
  • HTML — preferred (most AI-friendly)
  • Markdown (.md) — also works well
  • LaTeX (.tex) — acceptable
  • PDF — must be converted to markdown first. Ask the AI agent: "Convert this PDF to markdown: [file path]"
Tip: The demo used an HTML file: pre1workDemo/papers/KV_flow_paper_full.html. You can save web-based journal articles as HTML directly from your browser (File > Save As > HTML).

Step 2: Generate Your Outline — Send the File Path to AI

This is how you interact with the AI agent: you give it a file path and it reads the file and follows the instructions inside.

1 Send this file path to your AI agent:

pre1/teachingAssessment/pre1work/genOutline.md

The AI agent will:

  1. Read the assessment requirements (rubric, presentation brief, Session 4 handout)
  2. Read your paper in pre1work/papers/
  3. Generate two files:
    • outline_generated.md — your 8-minute presentation outline with presenter notes
    • slide_plan.md — a plan for your HTML presentation slides
Demo reference: See pre1workDemo/outline_generated.md and pre1workDemo/slide_plan.md for what the output looks like.

Step 3: Review the Outline and Write Feedback

Carefully read outline_generated.md and think about:

  • Is the content accurate to your paper?
  • Is the language accessible to non-specialists?
  • Are the writing lessons in Section 4 meaningful? (Can they be strengthened?)
  • Do you want specific visual elements on the slides?
  • Are there key quotes or evidence you want highlighted?

2 Write your feedback in pre1work/feedback.md, then send the file path to the AI agent:

pre1/teachingAssessment/pre1work/feedback.md

The AI will read your feedback and generate revised/additional files. Depending on your instructions, this may include:

updatedOutline.md→Revised outline incorporating your feedback and strengthening weak sections
style.md→Visual style guide (if you provided sample slides in sample_slides/)
updated_slide_plan.md→Revised slide plan combining your outline, feedback, and style preferences
slides4pre1.html→Your final HTML presentation slides
Demo reference: In the demo, the teacher's feedback in pre1workDemo/feedback.md asked the AI to: (1) convert sample .pptx files to markdown and extract styles, (2) strengthen the writing analysis section using course writing materials, (3) create updated slide plan, and (4) generate HTML slides. The AI produced 6 new files from that single set of instructions.

Step 4: Review and Revise Your Slides

Open the generated HTML file in your browser. Check:

  • Are the font sizes large enough? (See genHTMLguide.md for minimums)
  • Is the content enough for 8 minutes? (Aim for 10-12 content slides)
  • Does each slide have a slide number?
  • Are there transition cues between slides?
  • Do rubric labels appear on relevant sections?

If revisions are needed, create a reviseSlides.md file in your pre1work/ folder with your change requests, then send the path to the AI agent. The AI will generate a new version (e.g., slidesV2.html).

Demo reference: In the demo, the slides went through 3 versions: slides4pre1demo.html → slidesV2.html (added margins, slide numbers, rubric tags, editable mode) → slidesV3.html (larger fonts, better space usage). See pre1workDemo/reviseSlides.md for how the revision instructions were written.

Step 5: Edit Slides Directly in Browser

New feature: Your HTML slides are editable directly in the browser — a feature developed specifically for this course. You do not need any code editor to make last-minute changes to your slides.

How to use the built-in editing mode:

  1. Open your HTML slides in any browser (Chrome, Firefox, Safari, Edge)
  2. Press T to reveal the toolbar at the top of the screen
  3. Press E to enter edit mode — all text on the current slide becomes editable. Click any text to modify it directly.
  4. Use the toolbar buttons:
    • Save to Cache — saves your current edits to the browser's local storage (persists between sessions)
    • Load from Cache — restores a previously saved version
    • Download HTML — exports the edited slides as a new .html file you can save and share
    • Clear Cache — removes all saved data
  5. Press Escape to exit edit mode and return to presentation navigation
Tip: This means you can fine-tune wording, fix typos, and adjust content without going back to the AI agent. The downloaded file is a complete, self-contained HTML presentation — just like the original.

Step 6: Publish and Practice

  1. Commit and push all your files to your forked GitHub repository
  2. Your slides will be available via GitHub Pages at:
    https://[your-username].github.io/mccpSpring2026/pre1/teachingAssessment/pre1work/[your-slides].html
  3. Practice your delivery:
    • Review the Delivery Tips (interactive guide) — 7 skill levels covering body language, eye contact, voice, and audience interaction
    • Reference guide: pre1/teachingAssessment/deliveryTips.md
    • Practice out loud at least 3 times. Time yourself — aim for exactly 8 minutes
    • Record yourself and listen back for filler words and pacing issues

Step 7: Post to the Forum

Post a reply to the course Moodle forum with:

  • Your paper title and why you chose it
  • A link to your slides (GitHub Pages URL or your forked repo)
  • One thing you learned from the AI-assisted preparation process

Peer Feedback Guide — What to Watch For

During the 3-minute practice, your role as a peer reviewer is to give specific, actionable feedback on delivery (not content). Use this framework:

AreaWhat to observeFeedback starters
Eye contact Does the speaker look at the audience or stare at slides/notes? Do they cover all parts of the room? "I noticed you looked at the screen during... Try picking 3 anchor points in the room."
Body language Stable stance? Open posture? Purposeful gestures or fidgeting? "Your gestures were effective when... You could try keeping your hands still between points."
Voice Audible? Varied tone? Rushing or too slow? Filler words (um, uh, so)? "Your pace was good during the opening. Try pausing after key findings for emphasis."
Interaction Does it feel like the speaker is talking TO you or AT you? Any questions or engagement cues? "The rhetorical question at the start was great. You could add one more before the results section."
Overall presence Does the speaker appear confident and natural? Any script-reading? "You seemed most natural when... For the technical section, try using simpler language."
Tip: Good feedback is specific ("When you explained the methodology, you spoke faster") rather than vague ("Good job"). Mention one strength and one area to improve.

After the Session: Record a Full 8-Minute Presentation

After the preparation session, go home and record a full 8-minute practice presentation. This recording will be used for self-review and AI-assisted analysis.

Recording Instructions

  1. Set up a screen recording that captures both your slides and your face/voice:
    • Mac: Use QuickTime (File > New Screen Recording) or Zoom (record a meeting with yourself, share screen)
    • Windows: Use OBS Studio (free), Xbox Game Bar (Win+G), or Zoom
    • Simplest method: Start a Zoom meeting alone, share your slides screen, turn on your camera, and hit Record
  2. Open your HTML slides in full screen (press F11 or Cmd+Shift+F)
  3. Deliver the full 8-minute presentation as if presenting to a live audience:
    • Stand up if possible — it improves delivery
    • Look at the camera (your "audience"), not just the screen
    • Use the arrow keys to advance slides while speaking
  4. Save the recording as an MP4 or MOV file

Save the Recording to Your Repository

  1. Create a folder in your working directory: pre1work/recordings/
  2. Save your video file there (e.g., pre1work/recordings/practice_recording.mp4)
  3. Commit and push to your forked repository
File size note: Video files can be large. If your recording exceeds GitHub's 100MB file limit, you can either: (a) compress it using HandBrake (free) or an online video compressor, (b) upload it to a cloud service (Google Drive, OneDrive) and put the sharing link in a recording_link.md file instead, or (c) use Git LFS (ask your AI agent to help set this up).
