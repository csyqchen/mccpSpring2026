Important Notes

Deadline: Please complete this task by 15 February to allow sufficient time for the writing task (first draft by the end of February).

Scale of Analysis: The analyses can be done on a few files or more (e.g., 50-100 papers). Whether a larger scale analysis can reveal more insights depends on various factors—we need to review it case by case.

Submission Format: In your Moodle reply, include:

  1. The GitHub file path to your HTML analysis file (e.g., PhDagentSpring2026/literature/[YourFolderName]/task_achievement_analysis.html)
  2. A brief reflection on how the analysis informs your writing

Example submission format:

File path: PhDagentSpring2026/literature/[MyName]/task_achievement_analysis.html

Reflection: I analyzed two model papers from my target conference focusing on Task Achievement (Aspects 1 and 2). The analysis revealed that both papers use evaluative language strategically in centrality statements, and they employ different but equally effective strategies for gap identification—one uses systematic enumeration while the other uses concrete examples. This has helped me understand what "extremely clear" research background and "clearly delineated" gaps look like in practice. I plan to apply these insights by incorporating similar linguistic patterns and gap-signaling strategies in my own introduction and literature review.
